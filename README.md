# mktrial-guide

## 项目简介

`mktrial-guide` 是一个用于归档和发布独立 HTML 页面的仓库。这些页面通常用于展示特定功能、演示交互效果或提供临时指引。本仓库不针对任何单个域名或具体网站，而是作为一个集中存放与管理 HTML 资源的通用空间。

## 目录说明

```
mktrial-guide/
├── index.html          # 主入口页面（如有）
├── pages/              # 存放各独立 HTML 页面
│   ├── example1.html
│   ├── example2.html
│   └── ...
├── assets/             # 公共资源（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- **pages/**：包含所有独立的 HTML 页面，每个页面可自包含样式与脚本。
- **assets/**：存放页面间共享的静态资源，便于统一维护。
- **index.html**（可选）：可作为仓库的访问入口或导航页。

## 页面归档说明

- 每个 HTML 页面应独立完整，尽量减少对外部资源的依赖。
- 如需引用公共资源，请使用相对路径指向 `assets/` 目录。
- 页面命名应简洁明了，避免使用特殊字符或空格。
- 建议在页面头部添加简单注释，说明用途与创建日期。

## 维护说明

- 本仓库由维护者定期整理与更新。
- 新增页面时，请将其放入 `pages/` 目录，并更新本 README 的目录结构（如有需要）。
- 废弃页面可移入 `archive/` 目录（若存在）或直接删除。
- 如发现页面错误或链接失效，欢迎提交 Issue 或 Pull Request。

## 使用方式

1. 克隆或下载本仓库到本地。
2. 直接在浏览器中打开 `pages/` 下的 HTML 文件，或部署到静态托管服务。
3. 可根据需要修改、替换或新增页面内容。
